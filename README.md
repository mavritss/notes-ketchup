# Notes Ketchup

MVP Windows-приложения для быстрых заметок в Obsidian vault.

## Цель MVP

Сделать небольшое desktop-приложение для Windows, которое постоянно отображает окно поверх всех остальных окон и позволяет быстро сохранить заметку в тестовое Obsidian-хранилище:

```text
D:\Obsidian\notes-ketchup
```

Первая версия сознательно не использует AI, Git, синхронизацию, глобальные хоткеи и автоматическую сортировку. Главная задача MVP - надежно захватить текст и прикрепленные файлы, ничего не потерять и создать обычную Markdown-заметку, которую Obsidian сможет сразу увидеть.

## Что должно получиться

Приложение показывает компактное окно в стиле приложенного дизайна:

- большое поле ввода с плейсхолдером `Напишите что-то..`;
- нижняя панель с кнопками:
  - микрофон;
  - прикрепить файл;
  - скетч;
  - отправить;
- мягкий светлый контейнер с крупными скруглениями;
- окно всегда поверх других окон;
- окно остается открытым, пока приложение запущено;
- приложение работает независимо от Obsidian: Obsidian может быть закрыт.

## Функции первой версии

### 1. Текстовая заметка

Пользователь вводит текст в поле и нажимает кнопку отправки.

Приложение создает новую `.md` заметку в:

```text
D:\Obsidian\notes-ketchup\Inbox
```

Имя файла содержит дату и время создания:

```text
2026-06-25 15-40-12.md
```

Содержимое заметки:

```markdown
---
created: 2026-06-25T15:40:12+05:00
source: notes-ketchup
status: inbox
---

# 2026-06-25 15-40-12

Текст заметки.
```

### 2. Прикрепление файлов

Пользователь нажимает кнопку вложений и выбирает один или несколько файлов.

При отправке заметки приложение копирует выбранные файлы в:

```text
D:\Obsidian\notes-ketchup\99 Assets\Notes Ketchup\YYYY-MM-DD
```

Имена файлов нормализуются и получают дату/время, чтобы избежать перезаписи:

```text
2026-06-25_154012_original-name.pdf
2026-06-25_154012_photo.png
```

В Markdown-заметку добавляется раздел:

```markdown
## Вложения

- [[99 Assets/Notes Ketchup/2026-06-25/2026-06-25_154012_original-name.pdf]]
![[99 Assets/Notes Ketchup/2026-06-25/2026-06-25_154012_photo.png]]
```

Изображения вставляются через `![[...]]`, остальные файлы - через обычные wiki-ссылки.

### 3. Сохранность данных

Если при сохранении произошла ошибка:

- текст в окне не очищается;
- список выбранных файлов не сбрасывается;
- пользователь видит понятное сообщение об ошибке;
- можно повторить отправку.

После успешного сохранения:

- поле ввода очищается;
- список выбранных файлов очищается;
- показывается короткий статус, что заметка сохранена.

## Что не входит в MVP

В первой версии не делаем:

- AI-организацию заметок;
- Git commit / push;
- глобальные горячие клавиши;
- Obsidian-плагин;
- сканирование существующего vault;
- распознавание содержимого PDF, изображений и документов;
- автоматическое изменение старых заметок;
- облачную синхронизацию;
- публикацию установщика.

## Стек MVP

Выбран Tauri 2, потому что исходная цель - легкое Windows-приложение с настоящим desktop-окном, локальной файловой системой и режимом поверх всех окон.

Планируемые зависимости:

- Tauri 2 - desktop-приложение и системная часть окна;
- Rust - backend-команды приложения, запись файлов и копирование вложений;
- TypeScript - frontend-логика;
- Vite - dev-сервер и сборка frontend;
- React или Svelte можно добавить позже, но MVP можно собрать на простом HTML/CSS/TypeScript;
- готовые Tauri-плагины:
  - `tauri-plugin-dialog` для системного выбора файлов;
  - `tauri-plugin-fs` для безопасной работы с файлами, если понадобится;
  - `tauri-plugin-shell` не нужен в MVP.

Rust должен быть установлен в системе:

```text
C:\Users\<user>\.cargo\bin
```

Если текущая PowerShell-сессия еще не видит `rustc` и `cargo`, можно временно добавить стандартный путь Rust:

```powershell
$env:Path = "C:\Users\tema mavrits\.cargo\bin;$env:Path"
rustc --version
cargo --version
```

Важно: для сборки Tauri на Windows может понадобиться Microsoft C++ Build Tools с компонентом MSVC linker. Rust уже установлен, но наличие `link.exe` еще нужно проверить перед сборкой production-версии.

## Локальность и offline

MVP работает полностью локально:

- заметки создаются на диске;
- вложения копируются на диске;
- интернет не нужен;
- Obsidian не обязан быть запущен.

Кнопка микрофона в MVP остается визуально в интерфейсе, но диктовка не реализуется до выбора локального speech-to-text решения.

Возможные готовые варианты для будущей offline-диктовки:

- `whisper.cpp` - локальная транскрибация, хорошее качество, нужна модель;
- `faster-whisper` - локальная транскрибация через Python, быстрее на подходящем железе;
- Vosk - полностью offline, проще, но обычно хуже качество для свободной диктовки.

Для следующего этапа предпочтительно рассмотреть `whisper.cpp`, потому что он локальный и не требует отправки аудио во внешний сервис.

## План работы

### Шаг 1. Каркас приложения

- Создать `package.json`.
- Добавить Tauri 2 и Vite.
- Создать Tauri-приложение.
- Создать окно размером примерно `792x712`, ориентируясь на приложенный скрин.
- Включить:
  - `alwaysOnTop`;
  - отсутствие стандартной рамки или максимально минимальную рамку;
  - прозрачный или аккуратный фон;
  - запрет случайного скрытия окна.

Критерий готовности: приложение запускается и показывает пустое окно поверх других окон.

### Шаг 2. Интерфейс попапа

- Создать HTML/CSS/JS интерфейс.
- Использовать SVG-иконки из папки `icons`.
- Сделать большое поле ввода.
- Сделать нижнюю панель кнопок.
- Сделать кнопку отправки справа.
- Добавить состояния:
  - idle;
  - saving;
  - success;
  - error.

Критерий готовности: интерфейс визуально похож на приложенный скрин и не разваливается при изменении размера.

### Шаг 3. Настройки путей

В MVP путь vault можно зафиксировать в коде:

```text
D:\Obsidian\notes-ketchup
```

Папки:

```text
Inbox
99 Assets\Notes Ketchup
```

Критерий готовности: при первом сохранении приложение само создает нужные папки, если их нет.

### Шаг 4. Сохранение текстовой заметки

- По нажатию отправки собрать текст.
- Проверить, что текст или вложения не пустые.
- Создать Markdown-файл в `Inbox`.
- Добавить frontmatter.
- Добавить заголовок с датой.
- После успеха очистить форму.

Критерий готовности: новая заметка появляется в Obsidian vault даже при закрытом Obsidian.

### Шаг 5. Выбор и копирование вложений

- Открыть системный выбор файлов.
- Разрешить множественный выбор.
- Показать выбранные файлы в окне.
- При отправке скопировать файлы в assets-папку текущего дня.
- Добавить ссылки на вложения в Markdown.

Критерий готовности: заметка содержит рабочие Obsidian-ссылки на прикрепленные файлы.

### Шаг 6. Обработка ошибок

- Проверять доступность vault.
- Не очищать форму при ошибке.
- Показывать понятное сообщение.
- Не перезаписывать существующие файлы.

Критерий готовности: при ошибке пользователь может повторить сохранение без потери текста и списка файлов.

### Шаг 7. Подготовка следующего этапа

После MVP можно добавить:

- простую рисовалку для скетчей;
- offline-диктовку через `whisper.cpp`;
- страницу настроек vault;
- автозапуск приложения вместе с Windows;
- системный трей;
- упаковку в установщик.

## Ручная проверка MVP

1. Запустить приложение.
2. Убедиться, что окно поверх других окон.
3. Ввести тестовую заметку.
4. Прикрепить PDF и изображение.
5. Нажать отправку.
6. Проверить, что в `D:\Obsidian\notes-ketchup\Inbox` появился `.md` файл.
7. Проверить, что вложения скопировались в `99 Assets\Notes Ketchup\YYYY-MM-DD`.
8. Открыть vault в Obsidian.
9. Проверить, что заметка видна.
10. Проверить, что ссылки на вложения открываются.

## Запуск и сборка

Перед командами Tauri в обычной PowerShell можно добавить Rust в `PATH`:

```powershell
$env:Path = "C:\Users\tema mavrits\.cargo\bin;$env:Path"
```

Для надежной сборки Windows-приложения лучше запускать команды через окружение Visual Studio:

```powershell
cmd /c "call ""C:\Program Files\Microsoft Visual Studio\18\Community\Common7\Tools\VsDevCmd.bat"" -arch=x64 && set PATH=C:\Users\tema mavrits\.cargo\bin;%PATH% && npm.cmd run build"
```

Для разработки:

```powershell
cmd /c "call ""C:\Program Files\Microsoft Visual Studio\18\Community\Common7\Tools\VsDevCmd.bat"" -arch=x64 && set PATH=C:\Users\tema mavrits\.cargo\bin;%PATH% && npm.cmd run dev"
```

Готовый exe после сборки:

```text
src-tauri\target\release\notes-ketchup.exe
```

Готовый установщик:

```text
src-tauri\target\release\bundle\nsis\Notes Ketchup_0.1.0_x64-setup.exe
```
